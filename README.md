# RepoDigest Action

Generate and email a weekly engineering report from your GitHub activity — automatically.

[![GitHub Marketplace](https://img.shields.io/badge/Marketplace-RepoDigest-purple?logo=github)](https://github.com/marketplace/actions/repodigest)

## What it does

Every time it runs, RepoDigest:

1. Fetches commits, merged pull requests, and closed issues from your repository
2. Uses AI to generate a plain-English summary your stakeholders will actually read
3. Emails the report to everyone on your list

No dashboards. No logins. Just a clean weekly email.

## Quick start

```yaml
name: Weekly Engineering Report

on:
  schedule:
    - cron: "0 9 * * 1"  # Every Monday at 9am UTC
  workflow_dispatch:      # Allow manual runs

jobs:
  report:
    runs-on: ubuntu-latest
    permissions:
      contents: read
    steps:
      - uses: RepoDigest/repodigest-action@v1
        with:
          api_key: ${{ secrets.REPODIGEST_API_KEY }}
          recipients: "ceo@company.com, investor@fund.com"
```

## Setup

1. Sign up at [repodigest.com](https://www.repodigest.com)
2. Go to **Settings → API Keys** and create a new key
3. Add it to your repository as a secret named `REPODIGEST_API_KEY` (Settings → Secrets → Actions)
4. Add the workflow file above to `.github/workflows/weekly-report.yml`

## Inputs

| Input | Required | Default | Description |
|-------|----------|---------|-------------|
| `api_key` | ✅ | — | Your RepoDigest API key |
| `recipients` | ✅ | — | Comma-separated email addresses |
| `lookback_days` | ❌ | `7` | Days of activity to include |
| `api_url` | ❌ | `https://www.repodigest.com` | API base URL (for staging/self-hosted) |
